## Accomplishments

- Rewrote `project_session.py` — replaced `resolve(slug_or_name)` with `get_project_conn()` that takes no args, checks `DB_PATH.exists()` before opening, and exits with a clear error if not initialised
- Rewrote `init_project.py` — replaced `cmd_new/cmd_list/cmd_update` with `cmd_setup()` that:
  - Asks 5 click.confirm questions (sqlite_vec, otel, 3 gitignore entries)
  - Creates `project/`, `project/logs/`, `project/notes/` directories
  - Bootstraps `project/project.db` via `get_db(sqlite_vec_enabled=sqlite_vec)`
  - Writes `config/project.yaml` using `yaml.safe_dump` with `sqlite_vec` and `otel_enabled` keys
  - Appends idempotent `.gitignore` entries (static strings only)
  - Guards second run: exits 1 if `PROJECT_DIR` already exists
- Updated `test_init.py` — removed `TestParseJson` class (tests removed helper), removed xfail markers from tests that now pass
- Updated `test_setup.py` — removed xfail markers from all 9 tests that now pass; updated xfail reason on sqlite-vec test

## Decisions Made

- **get_project_conn() takes no args:** Strictly follows D-04 — no slug, no active sentinel. If DB_PATH missing, exits non-zero rather than creating an empty DB.
- **yaml.safe_dump only:** T-04-01 threat explicitly mitigated. Comment in source marks `yaml.dump` as forbidden.
- **gitignore static strings only:** T-04-02 threat mitigated. User's yes/no answer selects from a hardcoded list — no user text is interpolated.
- **test_setup_config_reflects_answers remains xfail:** Test passes `sqlite_vec=True` which requires sqlite-vec extension not installed in CI. Updated xfail reason to reflect this accurately.
