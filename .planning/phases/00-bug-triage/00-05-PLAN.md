## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| slug → file path | `C.db_path(slug)` constructs a filesystem path from the slug; used in `get_db()` |
| slug → SQL WHERE | `get_project_by_slug` uses parameterized query `WHERE slug = ?` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-00-05-01 | Injection | `get_project_by_slug(conn, slug)` SQL query | mitigate | Uses parameterized query `WHERE slug = ?` with positional binding — not string interpolation. SQLite3 stdlib handles escaping. |
| T-00-05-02 | Tampering | `upsert_project` slug auto-derive | accept | `C.slugify()` normalizes to `[a-z0-9-]` only, preventing special characters from entering the slug column. |
| T-00-05-03 | Information Disclosure | `ALTER TABLE` migration | accept | Migration runs on local SQLite files, not a shared DB. No data exposure risk. |
</threat_model>
